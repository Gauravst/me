Git Workflow Guide

1. Branching Strategy

Main Branches:

  • main → Stable production branch.
  • dev → Development branch where all features and fixes are merged.

Creating a New Branch

All new branches should be created from dev.

Feature Branches:

git checkout dev && git pull origin dev && git checkout -b feat/feature-name

Fix Branches:

git checkout dev && git pull origin dev && git checkout -b fix/bug-description

2. Commit Message Convention

Use clear and structured commit messages:

type: short description

Types of Commits:

  • feat → New feature (Example: feat: add login)
  • fix → Bug fix (Example: fix: resolve login issue)
  • docs → Documentation update
  • style → Code formatting
  • refactor → Code restructuring
  • test → Adding tests

3. Pushing Your Changes

git add . && git commit -m "feat: add new auth flow" && git push origin your-branch-name

4. Creating a Pull Request (PR)

1. Go to GitHub/GitLab.

2. Create a PR from your branch into dev.

3. Add a clear title and description.

4. Wait for approval before merging.

5. Merging to dev

Once approved, the PR is merged into dev. After testing, it will later be merged into main.

Summary

  • All branches are created from dev.
  • Use proper naming (feat/feature-name, fix/bug-name).
  • Follow commit message rules.
  • Always create a PR and wait for approval.
